Creación de Página Web

⭐️Pagina Del Grupo 604⭐️

Descubre cómo diseñamos esta increíble página web desde cero.

¿Cómo se construyó esta página?

Esta página fue desarrollada utilizando Visual Studio Code, HTML, CSS y GitHub Pages para publicarla en internet. Trabajamos en equipo para diseñar la estructura, el estilo visual y organizar el contenido, los cuales nos permitieron crear una página web completa desde cero. HTML se utilizó para estructurar todo el contenido, mientras que CSS se encargó del diseño visual, como colores, tamaños, posiciones y efectos. Desde el inicio del código, se incluye la etiqueta <!DOCTYPE html>, la cual indica al navegador que estamos trabajando con la versión más actual de HTML. Después se abre la etiqueta <html lang="es">, que define que todo el documento está en español,ademas los símbolos < > son fundamentales en HTML porque permiten identificar las etiquetas que estructuran una página web. Estos símbolos indican al navegador cuándo inicia y termina un elemento, como un párrafo, una imagen o un título. Todo el contenido que está dentro de estos signos es interpretado como una instrucción del código, por lo que el navegador lo convierte en elementos visibles o funcionales dentro de la página. Sin estos símbolos, HTML no podría distinguir entre texto normal y código, lo que haría imposible crear la estructura de un sitio web. Por esta razón, los símbolos < > son esenciales para definir la organización, el contenido y el funcionamiento de cualquier página web creada con HTMLl Dentro del <head> se encuentra información importante que no es visible directamente en la página. Aquí utilizamos <meta charset="UTF-8">, que permite que se puedan mostrar correctamente caracteres especiales como acentos y la letra ñ. También está <meta name="viewport" content="width=device-width, initial-scale=1.0">, que hace que la página se adapte correctamente a diferentes dispositivos como celulares o computadoras. La etiqueta <title> define el nombre que aparece en la pestaña del navegador. Además, dentro del <style> se encuentra todo el diseño de la página utilizando CSS.

En la parte del CSS, primero usamos *{ margin:0; padding:0; box-sizing:border-box; scroll-behavior:smooth; }, lo cual sirve para eliminar espacios automáticos del navegador, ajustar el tamaño de los elementos y permitir un desplazamiento suave al hacer clic en el menú. Después, en body definimos la fuente de letra, el color de fondo oscuro y el color del texto. También diseñamos el menú con header y nav, donde usamos position: fixed para que se quede siempre visible en la parte superior, y backdrop-filter: blur <8px> para darle un efecto moderno de desenfoque. El menú de navegación está formado por una lista <ul> con enlaces <a> que permiten moverse entre las secciones mediante los id, como #inicio, #construccion, etc. Gracias a esto, al hacer clic en una opción del menú, la página se desplaza automáticamente a esa parte. También se usaron estilos como display:flex para acomodar los elementos en línea y gap para separar los espacios entre ellos.

La página se divide en seis secciones principales utilizando la etiqueta <section>, lo que nos permitió organizar mejor el contenido. La primera sección, identificada con id="inicio" y clase "hero", funciona como la portada de la página. Aquí colocamos el título principal con <h1>, un subtítulo con <h2> y una pequeña descripción con <p>. Además, se utilizó una imagen de fondo con background:url(...) y se centró todo el contenido con display:flex, logrando una presentación visual atractiva.

La segunda sección, con id="construccion", es exactamente esta explicación que se está presentando, es decir, todo este texto forma parte de esa sección dentro de la página web. En esta parte usamos un título con clase "section-title" y un párrafo <p> donde describimos el proceso, las herramientas utilizadas como Visual Studio Code y el trabajo en equipo. Esta sección es importante porque aquí se explica directamente cómo fue desarrollado todo el proyecto.

La tercera sección, con id="imagenes", muestra las imágenes del proceso. Se utilizó un contenedor con clase "galeria" donde aplicamos display:grid para organizar las imágenes automáticamente en columnas. Cada imagen se agregó con la etiqueta <img> y se les dio estilo con border-radius para que se vieran con bordes redondeados, haciendo que la página se vea más moderna.

La cuarta sección, con id="videos", contiene los videos del proyecto. Aquí usamos la etiqueta <video> con el atributo controls, que permite reproducir, pausar y controlar el video directamente desde la página. También utilizamos un contenedor con clase "video-container" que funciona similar a la galería, organizando los videos en forma de cuadrícula.

La quinta sección, con id="referencias", incluye una lista de enlaces que utilizamos como apoyo. Se usó una lista <ul> con elementos <li> y dentro de ellos enlaces <a>. El atributo target="_blank" permite que los enlaces se abran en otra pestaña, evitando que el usuario salga de nuestra página.

La sexta y última sección, con id="integrantes", muestra los nombres de los miembros del equipo. También se utilizó una lista <ul> para organizar los nombres de forma clara y ordenada, permitiendo identificar quiénes participaron en el proyecto.

Aparte, dentro del desarrollo del proyecto se utilizaron diferentes propiedades y atributos de CSS y HTML que complementaron el diseño y la presentación del contenido. Por ejemplo, se aplicó la propiedad text-indent: 40px; dentro de la etiqueta p{} en el apartado de estilos, la cual permite generar una sangría al inicio de cada párrafo para darle una mejor presentación visual al texto. También se utilizó style="text-align: justify;" en el body, con el objetivo de justificar todo el contenido de la página y lograr una alineación uniforme tanto del lado izquierdo como del derecho. Asimismo, se implementó la inserción de imágenes dentro del texto utilizando la etiqueta , con el atributo src para indicar la ruta de la imagen y alt para describirla en caso de que no cargue correctamente. A esta imagen se le agregaron estilos en línea como float:right; para colocarla del lado derecho del texto, margin: 0 0 15px 20px; para generar espacio entre la imagen y el contenido, width:300px; para ajustar su tamaño y border-radius:10px; para darle bordes redondeados y un diseño más estético. Además, se utilizó style="margin-bottom: 30px;" en los párrafos para crear separación entre bloques de texto, mejorando así la organización visual de la página. En conjunto, estos elementos permitieron mejorar la estructura, legibilidad y presentación general del proyecto web.

Además, en este proyecto no se utilizó un archivo CSS externo, sino que todos los estilos se colocaron directamente dentro del mismo archivo HTML en la etiqueta <style>. Esto se hizo principalmente para facilitar el desarrollo y la organización del trabajo, ya que al ser un proyecto escolar y relativamente pequeño, resultaba más práctico tener todo en un solo archivo. De esta manera, evitamos manejar varios documentos y pudimos hacer cambios de forma más rápida y directa.

Trabajo en equipo El proyecto fue realizado colaborativamente organizando el contenido y revisando errores.

Para explicar la conformación de la sección dos, sobre sus elementos es importante centrarnos en los pasos que se realizaron para dar un toque especial y moderna a la página, siendo casi incluso interactiva.

Es fundamental analizar cómo combinamos la estructura de la etiqueta `section id="construccion"` con las reglas de estilo avanzadas que definimos en el encabezado del documento. Esta parte de la página no solo contiene la información sobre el proceso, sino que es un escaparate de técnicas de diseño moderno como el efecto parallax y el uso de contenedores con scroll interno.

Comenzamos nuestro desarrollo configurando la sección madre con el identificador `#construccion`, a la cual le asignamos un fondo dinámico mediante `background-image: url(...)`. Para lograr que la imagen parezca estar en un plano profundo mientras el usuario navega, aplicamos la propiedad técnica `background-attachment: fixed`, que fija la imagen de fondo mientras el resto del contenido se desliza, complementado con `background-size: cover` para que la fotografía siempre cubra el ancho y alto de la sección de forma proporcional. Dentro de este espacio, colocamos una capa visual llamada `div class="parallax-bg img-2"/div`, la cual refuerza esta profundidad mediante `position: absolute`, ocupando todo el ancho del contenedor con un `z-index: 1` para mantenerse en la base del diseño.

Sobre este fondo, construimos el elemento principal denominado `div class="capsula"`, que actúa como el contenedor del texto explicativo. En nuestro CSS, definimos para esta clase una anchura adaptable del `92%` y una altura fija basada en el visor del navegador con `height: 75vh`. Esta decisión técnica es clave, ya que al combinarla con la propiedad `overflow-y: auto`, permitimos que el contenido de texto largo se pueda leer cómodamente deslizándose dentro del cuadro gris sin que la página entera se mueva hacia abajo. Para la estética de esta "cápsula", utilizamos un fondo semitransparente con `background: rgba(20, 20, 20, 0.9)`, un borde sutil en color arena mediante `border: 1px solid rgba(245, 208, 169, 0.3)` y un efecto de desenfoque interno con `backdrop-filter: blur(10px)`, lo que le otorga una apariencia de cristal ahumado muy moderna.

Nos enfocamos también en los detalles del contenido interno, utilizando la clase `.section-title` para nuestro `h2`, el cual estilizados con un `font-size: 40px` y el color característico `#f5d0a9`. Para los párrafos `p`, aplicamos una lógica de lectura limpia mediante `line-height: 1.8` y el uso de `text-indent: 40px`, lo cual genera la sangría al inicio de cada bloque de texto como se especifica en nuestro estilo global. Además, decidimos personalizar la experiencia de usuario incluso en los elementos del sistema, por lo que incluimos reglas para los pseudo-elementos `::-webkit-scrollbar`, `::-webkit-scrollbar-track` y `::-webkit-scrollbar-thumb`. Con estas líneas de código, transformamos la barra de desplazamiento estándar del navegador en una versión delgada de `8px`, con un color arena que armoniza perfectamente con el resto de la interfaz oscura de nuestra página.

La tabla anterior que se logró visualizar fue gracias al comando "table" el cual nos permitió organizar la información de manera clara y estructurada, facilitando la lectura y comprensión de los elementos clave utilizados en el desarrollo de la página web. Cada fila de la tabla representa un elemento específico, logrando ser visualizado y estructurado por el código "tr" (la línea que define cada fila) y "td" (la línea que define cada celda dentro de la fila).

En conclusión, como equipo logramos desarrollar una página web bien estructurada, organizada en seis secciones claras, utilizando HTML para el contenido y CSS para el diseño. Durante el proceso trabajamos colaborativamente, revisando errores, probando el código y mejorando cada parte hasta obtener un resultado funcional, visualmente atractivo y fácil de navegar.
